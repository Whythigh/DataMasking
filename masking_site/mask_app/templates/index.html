HOW IT WORKS

From production data to
AI-ready data in seconds

Three steps. No configuration complexity. Integrates into your existing pipeline in under a day.

STEP 01

Send your data

POST a JSON payload or upload CSV / Excel / XML via the web UI. Define which fields to mask and how — name, IBAN, email, phone, amount, or full redaction.

→
STEP 02

Masking engine runs

Each field is processed by a format-specific masker. Referential integrity is preserved — the same real value always maps to the same fake value across all tables.

STEP 03

Get back structurally identical, statistically realistic data your AI can use without restrictions. Original data is never stored — deleted immediately after processing.

REST API

Integrate in
under an hour

A single POST endpoint. Send rows, get masked rows back. Drop it into any pipeline — Python, Node, Java, or curl.

python 
import requests

response = requests.post(
  "https://api.datarepli.com/v1/mask",
  headers={
    "Authorization": "Bearer dk_live_..."
  },
  json={
    "data": [
      {
        "name":   "Anna Tamm",
        "email":  "anna@seb.ee",
        "iban":   "EE38220012345...",
        "amount": 1247.50
      }
    ],
    "rules": {
      "name":   "mask",
      "email":  "mask",
      "iban":   "mask",
      "amount": "preserve"
    }
  }
)

print(response.json())
# → {"status":"done","rows":1,"masked_data":[...]}

How the API works

01

Authenticate with your API key in the Authorization header. Keys are tier-scoped — free, pro, business, or enterprise row limits.

02

Send your rows as a JSON array. Any shape — the engine maps rules to field names, not positions.

03

Define rules per field: mask for realistic substitution, redact for full removal, or omit to pass through unchanged.

04

Referential integrity is automatic within a job. Send multiple tables in one call and the same customer always maps to the same fake identity.
